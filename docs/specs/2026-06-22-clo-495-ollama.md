# Spec: Add Ollama local provider for zero-egress commits (CLO-495 / FR-56)

**Created**: 2026-06-22
**Estimated scope**: M (1 new file + ~6 touched, ~6 sub-tasks)
**Linear**: [CLO-495](https://linear.app/cloud-ai/issue/CLO-495) · **Slice** S8 · **Label** AFK · **Covers** FR-56
**Branch**: `feat/clo-495-ollama`

## 1. Problem Statement

`gcm` ships four provider paths behind the CLO-489 `Provider` trait (`src/provider/`): Groq and OpenAI (OpenAI-compatible chat, `src/provider/groq.rs` / `src/provider/openai.rs`), and Google Gemini (divergent shape, `src/provider/gemini.rs`). Every one of them sends the working-tree diff to a third-party HTTP API and requires an API key. There is **no zero-egress option**: a privacy-sensitive repo has no way to generate commits without the diff leaving the machine.

FR-56 (PRD `docs/prds/prd-gcm.md:131`) and ADR-001 Decision 8 (`docs/adrs/001-foundational-architecture-decisions.md:176`) require a **local Ollama provider** that talks to a local daemon (default `http://localhost:11434`, overridable via `OLLAMA_HOST`), needs **no API key**, uses a **user-selected local model**, and surfaces a **clear, actionable error when the daemon is unreachable**. This is the privacy anchor for the whole privacy story (FR-48–50): with a local model, nothing leaves the machine.

The architecture is **already decided** in ADR-001 Decision 8 and Appendix A (capability matrix) — this task implements that decision, it does not re-decide it:
- Use Ollama's **native `/api/chat`** endpoint with a JSON-Schema object in the `format` field (first-class structured output; no OpenAI-envelope translation). The OpenAI-compatible `/v1/chat/completions` path exists but is **not** preferred.
- The non-streaming `/api/chat` response is a single object with a top-level `message`; read `message.content`, **ignore `message.thinking`** (Ollama separates reasoning into its own field).
- Structured-output fidelity is **model-dependent** → rely on defensive parsing (`crate::plan::parse_defensive`, FR-20) + the existing `<think>` strip backstop; no strict-schema guarantee.
- No API key; honor `OLLAMA_HOST`; alias `gcml` (the shell alias itself is delivered by CLO-497, not here).

This task adds a fifth backend, `src/provider/ollama.rs`, implementing `Provider`, plus its registration and a small change to the shared transport so a backend can send **no** auth header.

### Key facts established by codebase probe

- `Provider` trait (`src/provider/mod.rs:27`): `name`, `generate_plan`, `generate_message`, `cache_model_id`, `diff_budget`. Synchronous (ADR-001 Decision 2).
- Selection registry (`src/provider/mod.rs`): `ProviderId` enum (`:148`), `default_model` (`:157`), `model_env_vars` (`:168`), `select()` match (`:186`), unknown-provider error list (`:225`, currently `"groq, google, openai"`).
- Shared transport `src/provider/http.rs`: `HttpRequest.auth` (`:38`) is a **mandatory** `(&'static str, String)` header pair; `send_once` (`:55`) always sets it. `post_json` retries only `RateLimit`/`Server` (FR-22). A dead endpoint → `map_ureq_error` (`:227`) → `ErrorKind::Transport(String)` (not retried) → Display "could not reach the {p} API: {msg}" (`src/provider/mod.rs:114`).
- `gemini.rs` is the closest template: own endpoint, own `extract_text` response parser, own payload builders, `auth: ("x-goog-api-key", key)`.
- `crate::plan::schema()` (`src/plan.rs:229`) returns a plain JSON-Schema object (lowercase types; used as the OpenAI `json_schema.schema`) — the correct helper for Ollama `format`. `gemini_schema()` (`:258`, OpenAPI-3.0 uppercase subset) is **not** used here. `crate::plan::parse_defensive` (`:80`) parses a plan JSON string defensively.
- Acceptance harness `scripts/acceptance.sh` already runs a python `http.server` mock on `127.0.0.1` that captures request bodies + auth headers and is pointed at via `GCM_<P>_BASE_URL`; a dead-port base (`http://127.0.0.1:9/...`, AC-12) is the established "unreachable provider" pattern. CI has **no** provider keys and **no** real Ollama daemon.

### Owner decision (classification checkpoint, 2026-06-22)

- **Default model `gemma4:e4b-mlx`** (local, Apple-Silicon MLX, zero-egress). `--model` / `GCM_OLLAMA_MODEL` override it.
- The owner also runs `deepseek-v4-flash:cloud` when on an Ollama Cloud subscription. Ollama's local daemon transparently proxies `*:cloud` models (after `ollama signin`) through the **same** `localhost:11434` endpoint, so gcm passes the model string unchanged and needs no separate endpoint or key. **Caveat:** a `:cloud` model egresses the diff to Ollama Cloud — it is **not** zero-egress. This is an explicit, documented opt-in via model override; it does not weaken the local default's guarantee.

## 2. Acceptance Criteria

- [ ] **AC-1 (zero-egress local commit)**: With a reachable Ollama daemon and a pulled local model, `gcm --provider=ollama` produces a signed commit, contacting **only** the configured local endpoint, with **no** API key set and **no** request to any third-party host.
- [ ] **AC-2 (unreachable → actionable error)**: With the daemon not running, `gcm --provider=ollama` exits non-zero, leaves the index untouched, and prints an error that names Ollama, names the endpoint, and tells the user how to fix it (start the daemon / set `OLLAMA_HOST`).
- [ ] **AC-3 (provider selectable, key-free)**: `--provider=ollama` and `GCM_PROVIDER=ollama` both select the backend with **no** API-key env var present; the unknown-provider error and `--help`/`--provider` text list `ollama`.
- [ ] **AC-4 (model selection)**: Model resolves by precedence `--model` > `GCM_OLLAMA_MODEL` > default `gemma4:e4b-mlx`; the resolved model is sent to the daemon and folded into the cache fingerprint as `ollama:<model>`.
- [ ] **AC-5 (endpoint config)**: Endpoint resolves by precedence `GCM_OLLAMA_BASE_URL` > `OLLAMA_HOST` > default `http://localhost:11434`; empty/whitespace values are treated as unset (matching the other providers). `OLLAMA_HOST` is normalized: a scheme-less value gets `http://` prepended, and a value with **no port** gets `:11434` appended (Ollama's default daemon port). Examples: `localhost` → `http://localhost:11434`; `127.0.0.1:8080` → `http://127.0.0.1:8080`; `https://host.example` → unchanged (no port forced on an explicit scheme/host). Requests go to `<base>/api/chat` (trailing slash on the base trimmed).
- [ ] **AC-6 (native structured output + reasoning hygiene)**: The plan request sends `"stream": false` and `"format": <plan schema>`; the response is parsed from `message.content` (defensively, FR-20), `message.thinking` is ignored, and no chain-of-thought reaches stdout. A response with empty `message.content` → `ErrorKind::EmptyResponse`; a response missing the `message` key entirely → `ErrorKind::Deserialize` (message names the missing `message` key).
- [ ] **AC-7 (missing-model → actionable error)**: When the daemon returns HTTP 404 for an unpulled model, the error (mapped to `ErrorKind::Config`, displayed verbatim) names the model and the endpoint and tells the user to `ollama pull <model>` (the default model is commonly not pre-pulled).
- [ ] **AC-8 (no auth header)**: No `Authorization` (or any auth) header is sent on any Ollama request; asserted by the acceptance mock capturing request headers and finding none.

**Verification method**: `cargo test` (unit tests in `mod.rs`, `ollama.rs`, `http.rs`); `bash scripts/acceptance.sh` (new AC-O cases against the localhost mock + dead-port); a manual real-Ollama run for AC-1/AC-6 end-to-end (Ollama is not in CI). `cargo fmt --check` + `cargo clippy --all-targets -- -D warnings` clean.

## 3. Constraints

**Must**:
- Implement the `Provider` trait in a new `src/provider/ollama.rs`; register it in `src/provider/mod.rs` (enum variant, `default_model`, `model_env_vars`, `select` arm, unknown-provider error list, `mod ollama;`).
- Use the **native `/api/chat`** endpoint (ADR-001 Decision 8), `"stream": false`, `format` = `crate::plan::schema()` for the plan call, no `format` for the single-message call.
- Parse the response from the top-level `message.content`; ignore `message.thinking`; apply the existing `strip_think` backstop. Empty content → `ErrorKind::EmptyResponse`; a body missing the `message` key entirely → `ErrorKind::Deserialize` naming the missing key (do not silently treat it as empty).
- Send **no** API key and **no** `Authorization` header. Make `HttpRequest.auth` optional (`Option<(&'static str, String)>`) and only set the header when `Some`; update `groq.rs`/`openai.rs`/`gemini.rs` to pass `Some(...)` and `ollama.rs` to pass `None`. The `Option` makes the compiler enforce that every present and future provider chooses `Some`/`None` explicitly. (Lower-risk alternative — sending a dummy `Authorization: Bearer ollama` — is rejected: the native endpoint defines no auth and a sentinel header is dishonest; the `Option` change is mechanical and keeps the transport truthful.)
- `HttpRequest.auth_env_var` (the separate `&'static str` used only to name the key in a 401/403 `Auth` message) is set to `""` for Ollama: it is unreachable code (no auth → no 401/403). `send_once` must reference `auth_env_var`/`auth` only on the `Some` path, so a `None`-auth request never reads the placeholder. Leave `auth_env_var` a plain `&'static str` (do not also make it optional) to keep the other providers untouched.
- Endpoint precedence `GCM_OLLAMA_BASE_URL` > `OLLAMA_HOST` > default `http://localhost:11434`; empty/whitespace treated as unset. Normalize `OLLAMA_HOST`: if it lacks a `://` scheme, prepend `http://`; if the host portion then has no explicit port, append `:11434` (Ollama's default). A value that already carries a scheme is taken as-is (no forced port). Trim a trailing slash before appending `/api/chat`.
- Default model `gemma4:e4b-mlx`; model env var `GCM_OLLAMA_MODEL`; `cache_model_id()` = `format!("ollama:{}", model)`.
- Remap two failures **inside `ollama.rs`** (do **not** widen the shared `ErrorKind` taxonomy), using the variant whose `Display` renders the message cleanly (verified against `src/provider/mod.rs:84`):
  - **Daemon unreachable (AC-2)** → re-wrap the `Transport(String)` from `post_json` with an enriched string: endpoint + "is Ollama running? start it with `ollama serve`, or set `OLLAMA_HOST`" + the underlying cause. `Transport`'s "could not reach the {p} API: …" prefix reads naturally and is truthful; it is already non-retryable, so the message appears immediately.
  - **Missing model 404 (AC-7)** → map `Http(404)` to `ErrorKind::Config(String)` (which `Display`s **verbatim**, no prefix/suffix), built from `self.model` + endpoint: "model '<model>' not found at <endpoint> (HTTP 404). Pull it with `ollama pull <model>`, or pick another with `--model` / `GCM_OLLAMA_MODEL`." `Http(404)` carries no string and `BadRequest` appends "…please report it" (implies a gcm bug), so neither can render this cleanly — hence `Config`. (Reviewer disagreement resolved this way after inspecting the real `Display` impls; parsing the daemon's own `{"error":"model '…' not found…"}` body is a possible future enrichment but is not needed and would require surfacing the 404 body through the shared transport.)
- Timeouts: the shared `GCM_HTTP_TIMEOUT_SECS` (default 60s, `http.rs`) applies to Ollama unchanged. Local MLX inference on a large diff can exceed 60s; document that raising `GCM_HTTP_TIMEOUT_SECS` is the remedy. Do not add a per-provider timeout in v1.
- `:cloud` disclosure: a `*:cloud` model (routed through the local daemon to Ollama Cloud) is **not** zero-egress. This must be disclosed in the CLI `EGRESS_DISCLOSURE` privacy text and the README provider section. (A runtime stderr warning on a `:cloud` model is a *Prefer*, below.)
- `--provider`/`GCM_PROVIDER` resolve `ollama` with no key (the `select`/resolve path stays key-free; the key — here, absence of one — is handled lazily inside the backend, matching the existing providers).
- Behavioral parity for the other four providers must be preserved (the `auth` change is the only shared edit; all existing tests must still pass).

**Must-not**:
- Must not require, read, or error on any API key for Ollama.
- Must not use the OpenAI-compatible `/v1/chat/completions` path or the `gemini_schema()` helper.
- Must not stream (omit/disable streaming with `"stream": false`; the tool needs the complete object).
- Must not add a new `ProviderId` clap alias for `gcml` (that shell alias is CLO-497); only the canonical `ollama` value.
- Must not introduce a new dependency or an async runtime (ADR-001 Decision 2).
- Must not claim zero-egress for `:cloud` models — document the egress caveat instead.

**Prefer**:
- `DiffBudget::standard()` for `diff_budget()` (mirrors Groq/Gemini; env-overridable per FR-13a) — local context windows vary, but the budget is overridable and a model-specific default is out of scope.
- `"options": { "temperature": 0.2 }` on both calls for determinism, matching the other backends.
- Omit the `think` field entirely (maximum cross-model compatibility: `message.content`/`message.thinking` separation already excludes reasoning; gpt-oss needs a level not a bool, so a blanket `think:false` is unsafe). A per-model `think` control is a future enhancement.
- Mirror `gemini.rs` structure (own parser, own payload builders, pure-function unit tests with no network).
- A one-line stderr note when the resolved model ends in `:cloud` (e.g. "note: '<model>' routes through Ollama Cloud; the diff is NOT zero-egress"), reinforcing the privacy story. Optional in v1 — the CLI/README disclosure is the required surface; a runtime warning is a nice-to-have the owner can confirm at the checkpoint.
- If an older daemon or a non-structured-output model rejects `format` with HTTP 400, the shared 400 path already surfaces the daemon's `error.message` via `BadRequest{detail}` — no special handling needed, but note this as the expected behavior (defensive parsing + the actionable 400 detail cover model-fidelity gaps, FR-20).

**Escalate when**:
- The native `/api/chat` non-streaming response shape or `format` semantics turn out to differ from ADR-001 Appendix A (verified 2026-06-19) in a way that breaks structured output — surface it rather than silently switching to the OpenAI-compatible path.
- Making `HttpRequest.auth` optional turns out to ripple beyond the 4 provider `request()` builders + `send_once` (it should not).

**Edge cases to verify**:
- Streaming default: omitting `"stream": false` would make `/api/chat` return NDJSON; the payload must force `false` (test 6/7). Must-not-regress.
- Port-less `OLLAMA_HOST` (`localhost`, `my-host.local`) must get `:11434` appended, else normalization yields port 80 and the connection fails (test 10).
- Trailing-slash `base_url` (`http://localhost:11434/`) must not double-slash `/api/chat` (trim like the other backends).
- `GCM_OLLAMA_BASE_URL` / `OLLAMA_HOST` empty/whitespace → treated as unset (matches the empty-env handling already used for models/providers).
- A `:cloud` model is just a model string (no special code path); it is documented as egressing — no test asserts egress, but the CLI privacy text + README must mention it (and the optional stderr warning, if added).
- Connection-refused / DNS failure / client timeout all surface as `Transport` (already non-retryable); the enriched AC-2 message is generic enough ("is Ollama running? … or set `OLLAMA_HOST`") to be actionable for each, and appears immediately rather than after the backoff budget.
- Local MLX inference on a large diff can exceed the 60s `GCM_HTTP_TIMEOUT_SECS` default → surfaces as `Timeout`; the remedy (raise the env var) is documented in README/CLI.
- Response missing the `message` key entirely → `ErrorKind::Deserialize` naming the key (test 9b), distinct from empty `message.content` → `EmptyResponse` (test 9).
- `message.content` present but not valid plan JSON → `parse_defensive` error surfaces as `Deserialize` (model-fidelity fallback per FR-20), not a panic.
- CI has no Ollama daemon: AC-O* run against the localhost mock only; the true end-to-end zero-egress AC-1 and native-structured-output AC-6 are a **manual** real-Ollama check (recorded in the implement-phase quality gate).
